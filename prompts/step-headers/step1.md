# Retrieval Step 1 — Pattern Results

**Contains:** Search-level results showing WHERE task-relevant code lives — which files match, which directories contain clusters, which patterns produce hits. Results come from refined, deduplicated searches derived from initial exploration.

**Does not contain:** File contents, code excerpts, or structural information (directory trees, file sizes). Those are produced by Steps 2 and 3.

**Downstream consumers:** Step 2 uses this file alongside a complete file listing to determine structural information needs. Steps 3 and 4 use it as a roadmap for what to read and where gaps may exist. Assembly uses it as part of the complete retrieval package.

Content appears in execution order, not logical order. Each entry is a command and its raw output.
